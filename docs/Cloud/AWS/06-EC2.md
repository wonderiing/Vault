`EC2` (Elastic Compute Cloud) es un servicio de AWS que nos permite rentar maquinas virtuales, nos permite almacenar datos, distribuir la carga y escalar los de manera automática. EC2 es un servicio a nivel de region.

## Lanzar Instancia

Código:

- El script básicamente actualiza el sistema, instala el paquete httpd y habilita el servicio httpd. Esto nos proporciona un servidor Apache HTTP por el puerto 80
- Después simplemente creamos un archivo HTML en la ruta estándar del servidor HTTP: `/var/www/html/index.html` (es importante ponerle index.html).

```
#!/bin/bash
yum update -y
yum install -y httpd
systemctl start httpd
systemctl enable httpd
echo "<h1>Hola mundo desde $(hostname -f)</h1>" > /var/www/html/index.html
```

## Tipos básicos de Instancias

AWS sigue la siguiente convención de nombres en las instancias

- m5.2xlarge

```
m -> Clase de instancia
5 -> Generacion de la instancia
2xlarge -> tamaño dentro de la clase de instancia.
```

**Instancia de Propósito General**:  Ofrecen un balance entre cómputo, memoria y red.

- **T (T3, T4g)**: rendimiento "ampliable" (burstable), muy económicas. Ideales para apps de bajo tráfico, entornos de desarrollo/pruebas o sitios administrativos.
- **M (M6i, M7g, M8g...)**: balance sólido. Buenas para aplicaciones como servidores web, repositorios de código y bases de datos pequeñas y medianas.[](https://aws.amazon.com/es/ec2/instance-types/)
- **A/Mac**: A para ARM de bajo costo; Mac para compilar/probar apps de Apple (iOS/macOS).

**Optimizadas para computación (Compute Optimized)**

- **C (C6i, C7g, C8id...)**: mayor relación vCPU/memoria. Ideales para procesamiento por lotes, transcodificación de contenido multimedia, servidores web de alto rendimiento, HPC, modelos científicos y motores de servidores de anuncios. [Amazon Web Services](https://docs.aws.amazon.com/es_es/ec2/latest/instancetypes/ec2-instance-type-specifications.html)
- También cubren modelado financiero, renderizado 3D y computación científica

**Optimizadas para memoria (Memory Optimized)**

- **R (R6i, R7g...)**: diseñadas para cargas de trabajo con uso intensivo de memoria, como bases de datos en memoria (Redis, Memcached). [IBM](https://www.ibm.com/es-es/think/topics/ec2-instance-types)
- **X**: memoria de muy alta capacidad, para SAP HANA, Apache Spark, big data.
- **z1d**: alta frecuencia de CPU + mucha memoria, para licencias por núcleo (bases de datos relacionales).
- **High Memory (u-...)**: instancias con terabytes de RAM, para SAP HANA a gran escala.

**Optimizadas para almacenamiento (Storage Optimized)**

- **I (I4i, I7ie)**: NVMe SSD local, para bases de datos NoSQL de alto IOPS (Cassandra, MongoDB), data warehousing.
- **D**: almacenamiento denso en HDD, para Hadoop/HDFS, data warehouses distribuidos.
- **H**: throughput secuencial alto para grandes volúmenes de datos.  
    Estas instancias se diseñaron para las cargas de trabajo que necesitan acceso de escritura y lectura secuencial alto a conjuntos de datos muy grandes en el almacenamiento local.

**Computación acelerada (Accelerated Computing)**

Usan GPU, FPGA o silicio propio de AWS.

- **P (P4, P5)**: GPU NVIDIA de alta gama, para entrenamiento de modelos de ML a gran escala.
- **G**: GPU para inferencia, gráficos y gaming en la nube.
- **Trn (Trainium)**: optimizadas para el entrenamiento distribuido de modelos de aprendizaje profundo, con interconexiones de alta velocidad, usando chips propios de AWS. [DataCamp](https://www.datacamp.com/es/blog/aws-ec2-instance-types)
- **Inf (Inferentia)**: perfectas para tareas de inferencia de baja latencia y alto rendimiento. [DataCamp](https://www.datacamp.com/es/blog/aws-ec2-instance-types)
- **F**: FPGA para cargas de trabajo personalizadas a nivel de hardware.

**HPC (High Performance Computing)**

- **Hpc7g, Hpc7a...**: ofrecen la mejor relación precio-rendimiento para ejecutar cargas de trabajo de HPC a escala, ideales para simulaciones complejas, deep learning y renderización de efectos visuales. [Amazon Web Services](https://aws.amazon.com/es/ec2/instance-types/)

## Grupos de Seguridad

Los grupos de seguridad son básicamente firewalls, bloquean o permiten trafico de entrada y salida.

Para crear un grupo de seguridad nos podemos ir a `EC2 -> Security Groups`

- Al crear un Security Group podemos ver las reglas de entrada y salida.
- Las reglas de entrada definen que puertos/servicios son accesibles por otras personas (trafico de fuera)
- Las reglas de salida definen los puertos/servicios a los que el propio servidor tiene acceso de entrar (trafico propio del servidor)

**Reglas de Entrada**

**Tipo:** El servicio o protocolo predefinido al que aplicará la regla (HTTPS, SSH, DNS, etc.). Si tu caso no está en la lista, eliges "Personalizado".

- **Protocolo:** TCP o UDP. Se autocompleta si elegiste un servicio predefinido; si es personalizado, lo eliges tú.
- **Intervalo de Puertos:** El puerto (o rango) por donde pasará el tráfico. Automático en servicios predefinidos (HTTPS=443, SSH=22); manual si es personalizado (puede ser un puerto único o un rango, ej: 8000-8100).
- **Origen:** Desde qué IP(s) se permite el tráfico entrante:
    - **Anywhere:** cualquier IP (0.0.0.0/0)
    - **My IP:** solo tu IP pública actual
    - **Personalizado:** una IP/rango específico (CIDR) u otro Security Group

**Reglas de salida:** Igual que las de entrada, pero para tráfico saliente. Solo permiten (no deniegan) — por defecto dejan pasar todo el tráfico. Para "bloquear" algún puerto como el 443, en realidad eliminas la regla "All traffic" y dejas solo las que sí quieres permitir (lo no permitido queda bloqueado).

## Conexión a una instancia

### SSH

Para conectarnos a través de SSH a una instancia es necesario lo siguiente:

- Par de claves (lo seleccionamos al crear una instancia). También puede ser creado desde `EC2 -> Par de Claves`
- El Security Group debe permitir el trafico entrante al puerto 22 SSH

![](assets/Pasted%20image%2020260710123716.webp)

Al crear la instancia elegimos una par de claves `ssh` en formato `pem`, esa clave la descargaremos y usaremos para conectarnos por SSH de esta manera:

!!! info "Mi par de claves se llama id_rsa, si el tuyo se llama distinto entonces remplaza el nombre en el comando"

- Windows: Desde una PowerShell podemos utilizar el siguiente comando (necesitamos la ip de la instancia y el user por defecto ec2-user):

```bash
> ssh -i .\id_rsa.pem ec2-user@18.189.28.14                                                                                

[ec2-user@ip-172-31-33-238 ~]$ ls
[ec2-user@ip-172-31-33-238 ~]$ whoami
ec2-user
```

- Mac o Linux: Es ligeramente distinto por que tenemos que colocarle permisos a la llave SSH

```bash
> chmod 600 id_rsa
> ssh -i id_rsa ec2-user@18.189.28.14  

[ec2-user@ip-172-31-33-238 ~]$ ls
[ec2-user@ip-172-31-33-238 ~]$ whoami
ec2-user
```
